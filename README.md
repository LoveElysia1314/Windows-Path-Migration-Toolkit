# Windows Path Migration Toolkit

一个功能完整的 Windows 应用路径迁移和系统配置修复工具，提供图形界面支持，专为系统管理员和高级用户设计。

**[English](README.md) | 简体中文**

---

## 📋 功能概览

### 🚀 应用迁移
- **应用扫描**：自动扫描系统中已安装的应用（支持 HKLM、HKCU、32位、64位）
- **智能过滤**：按关键词、安装路径、应用类型等条件过滤
- **位数自动分流**：根据应用位数（x86/x64）自动分配到不同目标目录
- **迁移预览**：支持查看迁移计划和手工修改目标路径
- **一键修复**：迁移后自动修复注册表项、环境变量、快捷方式等相关配置

### 🔧 盘符修复
- **批量替换**：将旧盘符（如 `E:` → `D:`）批量替换到系统配置
- **全面支持**：修复范围包括注册表、环境变量、快捷方式
- **独立窗口**：从主菜单快速打开，专注于盘符修复任务

### 📊 操作记录与恢复
- **完整记录**：每次迁移/修复操作都按批次记录和保存
- **详细日志**：支持查看操作详情、复制摘要
- **灵活恢复**：支持单条或批量恢复先前的操作
- **安全清理**：失败项支持选择性删除，可在重启后继续清理

### 🔒 安全与可追溯
- **自动备份**：执行操作前自动生成备份目录
- **完整恢复**：详细的恢复流程确保系统安全
- **管理员要求**：程序会自动检测并请求管理员权限
- **操作审计**：所有操作都可追溯和可撤销

## 🚀 使用方法

### 快速开始

1. **启动应用**
   ```bash
   python main.py
   ```

2. **应用迁移流程**
   - 点击 **扫描应用**，程序会自动扫描系统已安装应用
   - 使用搜索框和过滤条件缩小范围
   - 选择要迁移的应用，点击 **迁移**
   - 预览迁移计划，可手工调整目标路径
   - 确认无误后执行迁移，程序自动修复系统配置

3. **盘符修复流程**
   - 点击菜单 **工具** → **盘符修复**
   - 输入旧盘符和新盘符（如 E: → D:）
   - 确认修复范围（注册表、环境变量、快捷方式）
   - 执行修复，程序自动更新所有相关配置

4. **查看和恢复操作**
   - 点击 **历史记录** 标签
   - 查看操作详情、复制摘要
   - 选中记录，点击 **恢复** 可撤销操作

### 常见使用场景

**场景 1：磁盘扩容后重新分配程序**
- 原配置：程序安装在 C: 盘，现想迁移到 D: 盘
- 使用"应用迁移"功能，设置目标目录为 D:\Program Files

**场景 2：硬件升级导致盘符改变**
- 原配置：程序路径为 E:\App，新硬件中为 D:\App
- 使用"盘符修复"功能，将 E: 替换为 D:

**场景 3：环境变量路径失效**
- 原配置：环境变量指向旧路径，导致应用无法启动
- 使用"应用迁移"或"盘符修复"自动修复环境变量

## 📦 打包为独立可执行文件

使用 PyInstaller 将应用打包为 `.exe` 文件：

```bash
# 使用默认配置打包
python src/build_package.py

# 自定义打包选项
python src/build_package.py --name MyApp --onefile --clean
```

**打包选项：**
- `--name NAME`：自定义应用名称（默认：WindowsPathMigrationToolkit）
- `--onefile`：打包为单个 .exe 文件（默认分离文件）
- `--clean`：打包前清理旧的构建输出

打包完成后，`dist/` 目录下会生成可执行文件，可直接运行，无需 Python 环境。

## 🔑 关键特性

### ✅ 可靠性
- 操作前自动备份，支持完整回滚
- 每个操作都有详细日志记录

### ✅ 易用性

- 多语言支持（中文）
- 智能搜索和过滤

### ✅ 安全性
- 自动提权检查
- 敏感操作确认提示
- 可选的备份保留策略

### ✅ 高效性
- 异步后台任务处理
- 快速应用扫描和缓存
- 大批量迁移支持
- 应用列表已优化为模型视图渲染（QTableView + QAbstractTableModel），在大列表刷新场景下响应显著提升

## ⚠️ 注意事项

> **本工具会修改系统配置（注册表、环境变量、快捷方式）。请在理解风险后使用，并优先在测试环境验证。**

- **备份数据**：使用前请备份重要数据和系统快照
- **测试验证**：先在非关键应用上测试
- **管理员权限**：某些操作需要管理员权限
- **系统还原**：Windows 系统还原点可帮助恢复
- **反病毒软件**：可能会因修改注册表而触发安全警告

## 🐛 故障排查

### 问题：程序无法启动
**解决方案：**
```bash
# 检查依赖是否正确安装
pip show PySide6 pywin32

# 重新安装依赖
pip install --force-reinstall PySide6 pywin32
```

### 问题：提示"需要管理员权限"
**解决方案：**
- 右键点击 `main.py` 或 `.exe`，选择"以管理员身份运行"
- 或使用命令行：`python -m main` (如配置了提权)

### 问题：迁移后应用无法启动
**解决方案：**
- 查看历史记录，点击"恢复"撤销操作
- 检查目标路径是否正确
- 查看日志文件获取详细错误信息
